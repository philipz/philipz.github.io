## About Philipz（鄭淳尹），開源文化擁抱者，於 2013 年共同成立 [Docker.Taipei](https://www.facebook.com/groups/docker.taipei/)，在接觸過 Docker 後，認為所有資訊相關人員都必須具備 Docker 操作技巧，尤其開源策略已成為軟體產業宣傳及行銷的手段，因此任何開源程式都可用 Docker 包裝，而官方的 [Docker Hub](https://hub.docker.com/) 網站上有各式各樣的 Docker 映像檔可供下載使用，之後甚至還有微軟 Windows 平台的映像檔。從此，不必再煩惱安裝設定等枝微末節的「Dirty Jobs」，縮短IT人員的學習時間，直接切入核心軟體。 而 Docker 這三年的變化，尤其是 Machine 和 Swarm，宣告正式進入分散式叢集時代，可以很迅速啟動一整群服務，以“精兵政策”來開發複雜系統了，不再像以前動不動就需要十幾個人，而且團隊裡還需不少技術達人一起分工合作才能設計實作和維運整個分散式系統。 Octocat is Github's logo and trademark and here is used to illustrate a fictional character (John Smith) that used the *{ Personal }* Jekyll Theme to build his website.
